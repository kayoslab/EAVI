# ADR: CPU-Side Curl-Noise Advection with Pre-Computed LUT (`src/visual/curlLUT.ts`)

**Status:** Accepted

**Context:** US-082 replaces GPU-driven curl displacement in the particleField vertex shader with CPU-side Euler advection through a pre-computed curl-noise vector field. The existing approach evaluates curl3 (finite-difference FBM) per-vertex per-frame on the GPU, which couples macro flow control to the shader and prevents per-particle lifecycle management (recycling, fade-in, attractor interaction). Moving advection to the CPU enables coherent streamline integration, particle recycling with smooth fade-in, and attractor-based large-scale structure — none of which are feasible in a stateless vertex shader.

**Decision:**

1. **New shared module `src/visual/curlLUT.ts`** exposing:
   - `CurlLUT` interface: a 3D grid (default 32³) of pre-computed curl vectors stored as a flat `Float32Array`, with world-space bounds and resolution metadata.
   - `buildCurlLUT(seed, resolution?)`: one-time construction using CPU simplex noise with seeded permutation table. Deterministic and reproducible per session seed.
   - `sampleCurl(lut, x, y, z)`: trilinear interpolation of the 8 surrounding grid cells (~20 multiplies per lookup), returning a divergence-free velocity vector.

2. **CPU/GPU ownership contract** for the particleField system:
   - **CPU owns macro flow**: bass energy scales `fieldStrength` which controls advection speed through the curl LUT. Attractor radial multipliers amplify curl magnitude near vortex cores.
   - **GPU owns micro detail**: `uTrebleEnergy` drives per-point size modulation, high-frequency jitter, and color shimmer in the vertex/fragment shaders.
   - `uBassEnergy`, `uDisplacementScale`, and the `curl3()` displacement block are **removed from the vertex shader**. The shader becomes a pass-through for CPU-uploaded positions plus treble-driven effects.

3. **New optional attribute `aAlpha`** (itemSize 1) registered in `shaderRegistry.ts` following the established `OPTIONAL_PARTICLEFIELD_ATTRIBUTES` pattern. Used for particle fade-in on respawn to prevent visual popping.

4. **TypeScript simplex noise implementation** required for LUT construction. Preference is to use a lightweight npm package (e.g., `simplex-noise`) rather than porting the GLSL implementation, per review guidance.

**Consequences:**
- The `CurlLUT` pattern (pre-computed vector field with trilinear interpolation) is reusable: future systems needing divergence-free flow (e.g., ribbon advection, fluid-like effects) can consume the same module without reimplementing noise evaluation.
- CPU advection adds per-frame cost (~3,300 iterations worst-case at 550 particles with sub-stepping), but trilinear LUT lookup is O(1) per particle vs. the GPU's per-vertex FBM evaluation. Net GPU load decreases.
- The LUT is static after init — it does not follow `evolution.ts` parameter drift. Field evolution comes from attractor drift and bass-modulated speed, not from recomputing the noise field.
- Existing GPU-side features (slow modulation via `uEnableSlowModulation`, pointer repulsion via `uEnablePointerRepulsion`) are preserved in the simplified vertex shader as they operate on screen-space/time axes orthogonal to the curl flow.
- If 32³ resolution proves too coarse near attractor cores, the LUT can be rebuilt at 48³ (~110k cells) within the same <200ms budget, or a secondary high-res LUT can cover attractor neighborhoods only.
