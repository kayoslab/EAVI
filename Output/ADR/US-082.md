**ADR: CPU-Side Curl-Noise Library (`src/visual/curlNoise.ts`)**

**Status:** Accepted

**Context:** US-082 replaces the stateless GPU-only curl displacement in particleField with stateful CPU advection along a divergence-free vector field. This requires simplex noise, FBM, and curl computation to run in TypeScript each frame, not just in GLSL. Previously, all noise computation lived exclusively in `noise3d.glsl` and ran on the GPU.

**Decision:** Create a new shared module `src/visual/curlNoise.ts` that ports the GLSL noise algorithms to TypeScript, exposing:
- `snoise3(x, y, z)` — 3D simplex noise (same algorithm as `noise3d.glsl`)
- `fbm3(x, y, z, octaves)` — fractional Brownian motion with amplitude halving
- `curlNoise3(x, y, z, octaves)` — divergence-free curl field via finite-difference gradients of FBM

The module uses identical algorithmic definitions (simplex permutation, FBM amplitude/frequency scaling, finite-difference epsilon) as the GLSL version to maintain visual consistency between CPU-advected base positions and GPU-applied detail displacement.

Systems that adopt CPU advection read positions from Float32Array, compute `curlNoise3()` per particle, write back advected positions, and set `needsUpdate = true` on the position BufferAttribute each frame. The vertex shader retains responsibility for detail displacement (treble jitter, pointer repulsion, breathing) on top of CPU-advected base positions.

**Consequences:**
- New shared utility available for future systems to implement CPU-side flow fields or advection without duplicating noise code.
- Introduces a CPU/GPU algorithm parity concern: changes to noise3d.glsl must be mirrored in curlNoise.ts (and vice versa) to avoid visual divergence between base positions and detail displacement.
- CPU advection cost scales with particle count × octaves per frame — quality tiers gate octave count (low=1, medium=2, high=3) via existing `noiseOctaves` config to manage budget.
- Position buffer upload each frame (~7KB for 600 particles) is negligible but establishes the pattern of per-frame CPU→GPU attribute transfer for advected systems.
- The particleField system becomes stateful (positions evolve over time) rather than stateless (GPU computes positions from base + time). The GeometrySystem interface is unchanged — statefulness is internal to the system's `draw()` implementation.
