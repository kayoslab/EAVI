# ADR: Parametric Surface Curves Module (`src/visual/systems/parametricCurves.ts`)

**Status:** Accepted

**Context:** US-083 replaces the current helical band geometry in ribbonField with long coherent ribbons lying on parametric surfaces (helicoid, Mobius strip, torus knot). The existing ribbonField generates points along simple helices with random perpendicular spread. The new approach requires structured evaluation of parametric surface equations with position, normal, and tangent outputs — shared math that is distinct from any single visual system.

**Decision:** Introduce a new shared module `src/visual/systems/parametricCurves.ts` exposing:
- A `sample(u, v, seed) -> { position, normal, tangent }` interface per surface family
- Three families: helicoid, Mobius strip, torus knot (with seeded coprime (p,q) pairs)
- Session seed selects family and seeds parameters via existing PRNG infrastructure

Additionally, add an `aCurveParam` float attribute (normalized u-parameter, 0-1 along ribbon length) to the `RIBBONFIELD_ATTRIBUTES` registry. This enables GPU-side length-aware deformation: bass drives coherent wave propagation along ribbons (not just radial expansion), treble drives per-point shimmer perpendicular to tangent.

New shader files `parametricRibbon.vert.glsl` and `parametricRibbon.frag.glsl` are created rather than modifying the existing `ribbonWarp` shaders, to avoid regression risk in other systems.

**Consequences:**
- The `sample()` interface is reusable: future visual systems can generate geometry on parametric surfaces without reimplementing the math.
- `aCurveParam` follows the established optional-attribute pattern (`uHasSizeAttr`, `uHasVertexColor`) — systems that don't use it are unaffected.
- The `createRibbonField(config)` factory signature is unchanged; no breaking changes to mode registration, quality extraction, or compound modes.
- Torus knot (p,q) pairs must be clamped to known-good coprime values to avoid degenerate geometry (NaN/Infinity from trigonometric edge cases).
- Mobius strip inherently provides the required self-intersection; helicoid and torus knot variants achieve depth overlap via seeded v-offset band placement.
